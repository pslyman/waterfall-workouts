# Waterfall Workouts - Copilot Instructions

## Project Overview
Waterfall Workouts is an Ionic/Angular/Capacitor mobile and web application that implements a unique workout tracking methodology using a waterfall/kanban board approach. Workouts are prioritized by how overdue they are, with the most overdue appearing at the top.

## Technology Stack
- **Framework**: Ionic 8.6+ with Angular 20+
- **Mobile**: Capacitor 7.4+ (Android support, no Cordova)
- **Language**: TypeScript 5.8+
- **Storage**: @ionic/storage-angular for local persistence
- **Build**: Angular CLI with Ionic toolkit
- **Testing**: Jasmine + Karma

## Architecture & Key Concepts

### Core Methodology
- **Waterfall/Kanban Approach**: Workouts flow like a waterfall board
- **Priority by Overdue**: Most overdue workouts appear first
- **Time-based Tracking**: Days since last completion drives priority
- **Set-based Progress**: Track individual sets within workouts

### Data Model
```typescript
interface workoutsInt {
  days: number;           // Max days between workouts
  name: string;           // Workout name
  sets: number;           // Total sets to complete
  reps: number;           // Reps per set
  weight: string;         // Weight with unit (kg/lbs)
  countdown: number;      // Duration in minutes
  originDate: number;     // Last completion timestamp
  setsDone: number;       // Current sets completed
  notes: string;          // User notes
}
```

### Key Features
1. **Dynamic Sorting**: Workouts sorted by `days / timeSinceLastDone` ratio
2. **Set Tracking**: Individual set completion within workouts
3. **Metric/Imperial**: Weight units (kg/lbs) with user preference
4. **Themes**: Light/dark mode with Android status bar integration
5. **Local Storage**: Persistent data using Ionic Storage
6. **Animations**: Smooth card animations using Angular Animations

## Development Guidelines

### When Adding Features
1. **Storage First**: Consider data persistence needs
2. **Mobile UX**: Design for touch interfaces
3. **Performance**: Minimize DOM operations, use trackBy functions
4. **Accessibility**: Include proper ARIA labels and semantic markup
5. **Error Handling**: Graceful degradation and user feedback

### Code Style
- Use Angular/Ionic conventions
- Prefer declarative over imperative code
- Use async/await over Promise chains
- Type everything with TypeScript
- Follow Angular style guide

### Testing Approach
- Unit tests with Jasmine/Karma
- E2E tests with Protractor
- Manual testing on Android devices/emulators

## Common Tasks & Patterns

### Adding New Workout Properties
1. Update `workoutsInt` interface
2. Add form fields in tab1.page.html
3. Update add/edit methods in tab1.page.ts
4. Consider storage migration if needed

### Storage Operations
```typescript
// Always use the StorageService

const data = await this.storageService.get(key);
```

Remember: This is a personal workout tracking app focused on simplicity and the unique waterfall methodology. Prioritize user experience, data persistence, and the core workflow of tracking overdue workouts.
