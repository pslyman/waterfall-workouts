# Claude Developer Context - Waterfall Workouts

## Project Identity
**Waterfall Workouts** is an Ionic/Angular/Capacitor mobile application implementing a unique workout tracking methodology where exercises are organized like a waterfall/kanban board, with the most overdue workouts prioritized at the top.

## Technical Architecture

### Core Stack
- **Frontend**: Ionic 8.6+ Framework with Angular 20+
- **Mobile Runtime**: Capacitor 7.4+ (Android only, no Cordova)
- **Language**: TypeScript 5.8+ with strict typing
- **Data Persistence**: @ionic/storage-angular (local storage)
- **Build System**: Angular CLI + Ionic Angular Toolkit
- **Testing**: Jasmine/Karma for unit tests, Protractor for E2E

## Domain Model & Business Logic

### Core Entity: Workout
```typescript
interface workoutsInt {
  days: number;           // Maximum days between sessions
  name: string;           // Workout identifier
  sets: number;           // Total sets to complete
  reps: number;           // Repetitions per set
  weight: string;         // Weight with unit suffix (kg/lbs)
  countdown: number;      // Session duration in minutes
  originDate: number;     // Last completion timestamp (milliseconds)
  setsDone: number;       // Current completed sets
  notes: string;          // User annotations
}
```

### Waterfall Methodology Algorithm
**Priority Calculation**: `days / getDifferenceBetweenTimes(originDate)`
- Lower ratios = higher priority (more overdue)
- Completed workouts (setsDone === sets) reset automatically
- New workouts start with `originDate` offset to appear as overdue

### Key Business Rules
1. **Overdue Prioritization**: Most overdue workouts appear first
2. **Set-based Progress**: Users complete individual sets, not entire workouts
3. **Auto-reset**: Completing all sets resets the workout timer
4. **Metric/Imperial**: Weight units stored with values (e.g., "25kg", "50lbs")
5. **Time Tracking**: Days calculated from midnight-to-midnight precision
